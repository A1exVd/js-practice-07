### Вопросы к заданию

**Что такое делегирование событий и зачем оно нужно?**  
Шаблон проектирования при котором один обработчик событий вешается на родительский элемент, а не на все дочерние элементы. Благодаря эффектам всплытия и погружения события вызывается при нажатии на дочерний элемент. 

**Чем event.target отличается от event.currentTarget?**  
Event.target тот элемент на котором произошло событие. Event.currentTarget элемент для которого назначено событие. 

**Что делает stopPropagation() и когда его использовать?**  
Предотвращает дальнейшее распространение события от дочернего элемента к родительскому (всплытие) или родительского к дочернему (погружение). 
Используется для предотвращения конфликта обработчиков ( у родителя и ребёнка есть обработчик одного события и надо, чтобы сработал только один)

**Чем addEventListener лучше onclick?**  
Позволяет определять для одного события несколько обработчиков. 
Можно настроить фазу (захват), вызвать только один раз (once:true) или сделать пассивным.

**Что такое { once: true } в addEventListener?**  
Обработчик события выполнится только  один раз, после чего он удалится.

**Почему removeEventListener требует именованную функцию?**  
Потому что необходимо передать ссылку на обработчик, который хотим удалить.
